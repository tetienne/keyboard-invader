# Phase 2 — UI Design Contract

> Visual and interaction contract for the game engine foundation. This phase is primarily infrastructure -- canvas rendering, game loop, input handling, state machine. Visual output is developer-facing (debug overlay) and placeholder (menu screen, falling rectangles). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Vanilla TypeScript + PixiJS) |
| Preset | not applicable |
| Component library | none (PixiJS for canvas, raw DOM for overlays) |
| Icon library | none (not needed this phase) |
| Font | BitmapFont "GameFont" (Arial base, installed via PixiJS BitmapFont.install) |

**Rationale:** This is a canvas-based game, not a component-driven web app. PixiJS handles all in-game rendering. Tailwind CSS (already installed) handles any DOM elements outside the canvas. No shadcn -- project uses Vanilla TypeScript per DX-05.

## Interaction Contract

### Keyboard Input (D-08 through D-11)

| Input | Context | Behavior |
|-------|---------|----------|
| Any letter key | Playing state | Destroys nearest matching placeholder object (or any object for engine demo) |
| Any key (repeat) | All states | Ignored -- only initial keydown processed (D-08) |
| Ctrl+X, Alt+X combos | All states | Ignored -- modifier combos filtered out (D-10) |
| F3 | All states | Toggle debug overlay visibility (D-18) |
| Click on "Jouer" | Menu state | Transition to Playing state (D-07) |

### State Transitions (D-04 through D-06)

| From | To | Trigger |
|------|----|---------|
| Boot | Menu | PixiJS app initialized, assets ready |
| Menu | Playing | User clicks "Jouer" |
| Playing | Paused | Window loses focus (blur event, D-17) |
| Paused | Playing | Window regains focus |
| Playing | Menu | all placeholder objects cleared (session end) |
| Paused | Menu | (optional, not required this phase) |

Invalid transitions throw errors in dev mode (D-05).

### Tab/Focus Behavior (D-16, D-17)

| Event | Behavior |
|-------|----------|
| window.blur | Pause game loop, show "PAUSE" text |
| window.focus | Resume game loop, hide "PAUSE" text, reset time accumulator (D-16) |
| document.visibilitychange (hidden) | Same as blur |
| document.visibilitychange (visible) | Same as focus |

## Layout Contract

### Canvas (D-14, D-15)

- Base resolution: 1280x720 (16:9)
- Scaling: proportional scale to fill window, maintaining 16:9 aspect ratio
- Letterboxing: black bars (#000000) on sides or top/bottom when window aspect does not match
- Canvas element fills `#game-container` div (100vw x 100vh)

### Debug Overlay (D-18, D-19)

- Position: top-left corner, 16px from viewport edge
- Background: rgba(22, 33, 62, 0.85), border-radius 4px, padding 8px
- Font: system monospace, 14px, color #a0aec0
- Z-index: above canvas (HTML DOM overlay, not PixiJS rendered)
- Hidden by default, toggled with F3
- Content layout: vertical stack of "Label: Value" lines

### Menu Placeholder (D-07)

- Rendered inside PixiJS canvas (not DOM)
- Game title: "Keyboard Invader" in BitmapText, 48px, centered horizontally, positioned at ~35% from top
- "Jouer" button: BitmapText, 24px, centered horizontally, positioned at ~55% from top
- "Jouer" text is interactive (PixiJS eventMode: 'static', cursor: 'pointer')
- Hover state: scale to 1.1x (simple transform, no color change needed for placeholder)

### Pause Overlay

- "PAUSE" text: BitmapText, 48px, white (#ffffff), centered on canvas
- Semi-transparent overlay behind text: rgba(0, 0, 0, 0.5) covering full canvas
- Rendered as PixiJS graphics + text (not DOM), part of the game state rendering
