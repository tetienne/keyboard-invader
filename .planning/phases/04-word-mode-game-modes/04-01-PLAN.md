    3. **getAvailableWords function (per D-12):** Same signature pattern as getAvailableLetters. Takes progress, total, and WordLists. Returns short words when progress/total < 0.4, returns short + medium when >= 0.4. Returns short if total is 0.
       ```typescript
       export function getAvailableWords(
         wordLists: WordLists,
         progress: number,
         total: number,
       ): readonly string[] {
         if (total === 0) return wordLists.short
         const ratio = progress / total
         if (ratio < 0.4) return wordLists.short
         return [...wordLists.short, ...wordLists.medium]
       }
       ```

    4. **WordEntity interface (per D-07):** Similar to LetterEntity but with word-specific fields.
       ```typescript
       export interface WordEntity {

         baseX: number
         originalTint: number
         tween: LetterTween | null
         markedForRemoval: boolean
       }
       ```
       NOTE: For the pure-logic module, define WordEntity with `text` typed as a generic display object (same pattern as TweenTarget in tween.ts). The actual SplitBitmapText type will be used in states.ts where PixiJS is available. Use:

    5. **findActiveWord function (per D-07):** Returns the lowest (highest y value) word entity that has no active tween and is not marked for removal. Same pattern as findLowestEntity.
       ```typescript
       export function findActiveWord(active: readonly WordEntity[]): WordEntity | null {
         let best: WordEntity | null = null
         for (const entity of active) {
           if (entity.tween === null && !entity.markedForRemoval) {
             if (!best || entity.text.y > best.text.y) {
               best = entity
             }
           }
         }
         return best
       }
       ```

    6. **matchWordKey function (per D-05, D-06):** Takes an active WordEntity and a key string. Returns 'correct' if key matches word[cursorIndex], 'complete' if that was the last character, 'wrong' otherwise. Does NOT mutate the entity; state mutation is the caller's job.
       ```typescript
       export type WordMatchResult = 'correct' | 'complete' | 'wrong'

       export function matchWordKey(entity: WordEntity, key: string): WordMatchResult {
         const expected = entity.word[entity.cursorIndex]
         if (key !== expected) return 'wrong'
         if (entity.cursorIndex >= entity.word.length - 1) return 'complete'
         return 'correct'
       }
       ```
