    **C. PlayingState: Mode-aware gameplay (per D-23, D-24)**

    This is the core change. PlayingState must handle both letter and word mode.

    1. Add new private fields for word mode:
       ```typescript
       private activeWordEntities: WordEntity[] = []
       private wordLists: WordLists | null = null
       private timePlayedMs = 0
       private mode: GameMode = 'letters'
       ```
       Import `WordEntity`, `WordLists`, `loadWordLists`, `getAvailableWords`, `findActiveWord`, `matchWordKey` from `./words.js`. Import `SplitBitmapText` from pixi.js. Import `getLocale` from `../shared/i18n/index.js`.

    2. Add word mode constants:
       ```typescript
       private readonly WORD_FALL_SPEED = 50       // D-04: slower than letter mode's 80
       private readonly WORD_SESSION_LENGTH = 15    // fewer words than letters per session
       private readonly WORD_SPAWN_INTERVAL_MS = 2500  // more time between spawns
       ```

    3. In `enter()`:
       - Read mode from `ctx.getGameMode()` and store in `this.mode`.
       - Reset `this.timePlayedMs = 0`.
       - If word mode: load word lists via `loadWordLists(getLocale())` and store in `this.wordLists`.
       - Use mode-appropriate SESSION_LENGTH and SPAWN_INTERVAL for the existing spawn timer logic.

    4. In `update()`:
       - Add `this.timePlayedMs += dt` at the top (per D-18, per Pitfall 5 from research: accumulate dt, not wall clock).
       - **Spawn logic branch:** If letters mode, use existing code. If words mode:
         - Use `getAvailableWords(this.wordLists!, this.totalSpawned, wordSessionLength)` to get word pool.
         - Pick a random word from the pool.
         - Call `ctx.acquireWordPoolItem()` to get a SplitBitmapText.
         - Cast item to SplitBitmapText. Set `.text = word.toUpperCase()`.
         - Call `.updateText()` on the SplitBitmapText to ensure `.chars` is populated (Pitfall 1 from research).
         - Reset all char tints to the chosen color (Pitfall 2).
         - Calculate width and constrain x position: `x = wordWidth/2 + Math.random() * (BASE_WIDTH - wordWidth)` to prevent edge overflow (Pitfall 4 from research).
         - Set y = -40, scale 1, alpha 1, visible true.
         - Push a WordEntity to `this.activeWordEntities`.

       - **Fall logic branch:** Words use `WORD_FALL_SPEED`, letters use `FALL_SPEED`. Iterate the correct entity array.

       - **Input processing branch (per D-05, D-06, D-07, D-08):**
         - Letter mode: existing findLowestMatch logic unchanged.
         - Word mode: call `findActiveWord(this.activeWordEntities)`. For each key from input buffer:
           - Call `matchWordKey(activeWord, key)`.
           - If 'correct': set `activeWord.text.chars[activeWord.cursorIndex].tint = 0x4ade80` (green), increment `activeWord.cursorIndex`, increment hits.
           - If 'complete': set last char green, trigger `createHitTween()` on the word entity, set `markedForRemoval = true`, increment hits.
           - If 'wrong': increment misses. Apply miss feedback: `createMissTween()` on the active word (red flash + shake). Important: per D-05, "no progress lost" so do NOT reset cursorIndex.

       - **Tween and cleanup:** Process BOTH `this.activeEntities` (letters) and `this.activeWordEntities` (words). Only one array will have items based on mode, so iterating both is safe and simple.

       - **Bottom detection:** Apply to word entities same as letter entities (y > BASE_HEIGHT + 40).

       - **Session end:** Check the mode-appropriate session length and entity array.
         On session end, call:
         ```typescript
         ctx.setSessionResult({
           hits: this.hits,
           misses: this.misses,
           total: modeSessionLength,
           timePlayed: this.timePlayedMs,
           mode: this.mode,
         })
         ```

    5. In `exit()`:
       - Release all word entities via `ctx.releaseWordPoolItem(entity.poolIndex)`.
       - Clear `this.activeWordEntities = []`.
       - Reset `this.wordLists = null`, `this.timePlayedMs = 0`.
