## Typography

All text on the PixiJS canvas uses BitmapText with `fontFamily: 'GameFont'`. Sizes below are PixiJS fontSize values (not CSS px). The canvas base resolution is 1280x720; PixiJS auto-scales.

| Role | Size | Weight | Line Height | Phase 7 Usage |
|------|------|--------|-------------|---------------|
| Display | 48px | normal (400) | 1.2 | "Bravo !" title (existing), "Niveau {n} !" celebration text |
| Heading | 28px | normal (400) | 1.2 | "Rejouer" CTA button (existing), XP earned summary "+42 XP" |
| Body | 22px | normal (400) | 1.4 | Results stats lines (existing), XP bar label "45/120" |
| Label | 18px | normal (400) | 1.4 | HUD level badge "Niv. 3", mini XP text, lock level label |
| Small | 14px | normal (400) | 1.4 | Lock tooltip message |

Note: BitmapFont does not support font weights. All text is rendered at normal weight. Visual emphasis is achieved through size, color, and tint instead.

### XP Bar Gradient (layered solid fills)

PixiJS v8 Graphics does not support gradient fills. The XP bar uses two overlapping rounded rectangles to simulate a gradient effect.

| Layer | Color | Purpose |
|-------|-------|---------|
| Background | 0x2d2d4a | Dark pill outline, full width |
| Fill (left edge) | 0x4dabf7 | Blue start of fill |
| Fill (right edge) | 0xa78bfa | Purple end of fill |

Implementation: draw two overlapping fill rects (left half blue, right half purple) clipped by the fill width. Alternatively, use a single fill color 0x6b8bf5 (blue-purple midpoint) for simplicity. The executor should choose whichever yields acceptable visual results with less complexity.

## Interaction Contracts

### XP Bar Fill Animation (Results Screen)

| Property | Value |
|----------|-------|
| Trigger | GameOverState enters, after stats display |
| Delay | 300ms after entering results (let stats appear first) |
| Duration | Proportional: 800ms for full bar fill, minimum 200ms for small fills |
| Easing | easeOutQuad: `t * (2 - t)` |
| On level-up | Bar fills to 100% -> 200ms pause -> celebration overlay -> 500ms pause -> bar resets to 0 at new level -> continue filling |

### Celebration Overlay Sequence

| Step | Duration | Action |
|------|----------|--------|
| 1 | 0-600ms | Level text scales in with bounce |
| 2 | 0-200ms | Particle burst spawns (concurrent with step 1) |
| 3 | 200-2000ms | Particles drift with gravity + fade |
| 4 | 2500ms | Auto-dismiss overlay |

### Locked Avatar Tap

| Property | Value |
|----------|-------|
| Trigger | Player taps a locked (grayed) avatar on profile screen |
| Response | Show encouraging BitmapText tooltip below avatar for 2s, then fade out |
| Text | "Atteins le niveau {n} pour debloquer !" at 14px, muted color |
| Selection | Locked avatars are NOT selectable for profile creation or editing |

### HUD XP Bar (PlayingState)

| Property | Value |
|----------|-------|
| Visibility | Always visible during gameplay, top-left corner |
| Updates | Render current level progress from profile data (static during session) |
| Interaction | None (display only, no tap/hover) |
