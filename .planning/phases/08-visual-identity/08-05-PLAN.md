  <name>Task 1: Lower bottom detection threshold so escape tweens trigger on-screen</name>
  <files>src/game/states.ts</files>
  <action>
The root cause is the bottom detection threshold, NOT the tween completion handler.

Currently (around lines 562-584), bottom detection triggers at:
```typescript
entity.container.y > BASE_HEIGHT + 40
```

BASE_HEIGHT is 720. So detection fires at y > 760 — the entity is already 40px below the visible viewport. The escape tween then animates alpha, scale, and x (but NOT y), so the entire 600ms animation plays off-screen and is never seen.

**Fix the bottom detection threshold** in BOTH loops (letter entities around line 566 and word entities around line 578):

Change:
```typescript
entity.container.y > BASE_HEIGHT + 40
```
To:
```typescript
entity.container.y > BASE_HEIGHT - 60
```

This triggers the escape tween when the entity reaches y = 660, which is 60px from the bottom of the visible 720px viewport. The entity is still fully visible, giving the 600ms escape animation (fade + shrink + lateral drift) time to play while the player can see it.

**Also** add 'escape' to the tween completion handler type check in both `_updateTweens` (around line 855) and `_updateWordTweens` (around line 871). Change:
```typescript
if (entity.tween.type === 'miss' || entity.tween.type === 'dodge') {
```
To:
```typescript
if (entity.tween.type === 'miss' || entity.tween.type === 'dodge' || entity.tween.type === 'escape') {
```

This resets x position after the escape tween completes (the entity drifts laterally during escape, so baseX reset keeps cleanup consistent with miss/dodge).

Summary of changes — two locations, two changes each:
1. Bottom detection threshold: `BASE_HEIGHT + 40` to `BASE_HEIGHT - 60` (both letter and word loops)
2. Tween completion handler: add `'escape'` to the type check (both `_updateTweens` and `_updateWordTweens`)
