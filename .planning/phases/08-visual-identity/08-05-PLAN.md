  <files>src/game/states.ts</files>
  <action>
In `_updateTweens()` (around line 855) and `_updateWordTweens()` (around line 871), the tween completion handler currently checks:

```typescript
if (entity.tween.type === 'miss' || entity.tween.type === 'dodge') {
  entity.container.x = entity.baseX
  entity.tween = null
} else {
  entity.tween = null
}
```

The 'escape' tween type falls into the else branch, which nullifies the tween immediately. The cleanup pass then removes the entity (markedForRemoval=true and tween=null) before the escape animation is visible.

Fix: Add 'escape' to the condition in BOTH methods so escape tweens follow the same completion path as miss and dodge:

```typescript
if (entity.tween.type === 'miss' || entity.tween.type === 'dodge' || entity.tween.type === 'escape') {
  entity.container.x = entity.baseX
  entity.tween = null
} else {
  entity.tween = null
}
```

This ensures the escape animation runs its full 600ms duration (fade + shrink + lateral drift) before the entity is cleaned up, matching the behavior of dodge and miss tweens.

Apply this change in both `_updateTweens` (letter entities) and `_updateWordTweens` (word entities). The two methods have identical tween completion logic -- both need the same fix.
