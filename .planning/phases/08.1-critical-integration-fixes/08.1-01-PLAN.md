**Fix 2 -- Wire alien idle animation (per D-05, D-06):**

In src/game/states.ts, inside `PlayingState.update()`, add updateIdle calls in the existing entity iteration loops. The best placement is in the fall logic section (around lines 526-535) since we already iterate both arrays there. Add the updateIdle call for EVERY entity (not just non-tweening ones, per D-06):

Find the fall logic loops and add updateIdle to each:

FOR activeEntities loop (around line 526-530), change:
```typescript
    for (const entity of this.activeEntities) {
      if (entity.tween === null && !entity.markedForRemoval) {
        entity.container.y += fallSpeed * dtSec
      }
    }
```
TO:
```typescript
    for (const entity of this.activeEntities) {
      entity.container.updateIdle(dt)
      if (entity.tween === null && !entity.markedForRemoval) {
        entity.container.y += fallSpeed * dtSec
      }
    }
```

FOR activeWordEntities loop (around line 531-535), change:
```typescript
    for (const entity of this.activeWordEntities) {
      if (entity.tween === null && !entity.markedForRemoval) {
        entity.container.y += fallSpeed * dtSec
      }
    }
```
TO:
```typescript
    for (const entity of this.activeWordEntities) {
      entity.container.updateIdle(dt)
      if (entity.tween === null && !entity.markedForRemoval) {
        entity.container.y += fallSpeed * dtSec
      }
    }
```

The `updateIdle(dt)` call is OUTSIDE the `if` guard, so it runs on ALL entities -- including those currently running tweens (per D-06). The tween system modifies container-level properties (x, alpha, scale) while idle modifies sprite-level properties (sprite.y, sprite.scale.y), so there is no conflict.
