# Phase 08.1 Plan 01: Critical Integration Fixes Summary

Three surgical bug fixes closing integration gaps found by the v1.0 milestone audit: avatar ID mismatch in persistence defaults, missing difficulty signal in word mode, and unwired idle animation method in the game loop.

## Tasks Completed

### Task 1: Fix DEFAULT_UNLOCKED_AVATARS to use current avatar IDs
- Replaced legacy avatar IDs (`avatar-red`, `avatar-blue`, `avatar-green`) with current IDs (`avatar-kid-01`, `avatar-kid-02`, `avatar-alien-01`) in `src/persistence/types.ts`
- Added cross-validation test ensuring all default avatar IDs exist in the AVATARS definitions array
- Fixed pre-existing ESLint issues in `tests/persistence/schema.test.ts` (`as any` -> `as unknown as ProfileData[]`, `!` -> optional chaining)

### Task 2: Add word mode miss signal and wire alien idle animation
- Added `this.difficulty.recordResult(false)` to `_processWordInput()` wrong-key branch, matching the letter mode pattern
- Wired `entity.container.updateIdle(dt)` in both `activeEntities` and `activeWordEntities` fall loops in `PlayingState.update()`
- `updateIdle` runs outside the tween guard, so all entities bob and blink continuously (no conflict with tween system)
- Added 3 new tests using DifficultyManager prototype spy and mock `updateIdle` functions
- Fixed pre-existing ESLint issues in `tests/game/states.test.ts` (array-type, naming-convention, empty-function, void-expression)
