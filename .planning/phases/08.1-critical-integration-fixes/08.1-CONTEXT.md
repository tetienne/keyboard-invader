# Phase 8.1: Critical Integration Fixes - Context

**Gathered:** 2026-04-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix three integration defects found by the v1.0 milestone audit: avatar ID mismatch breaking new profile creation, missing difficulty signal in word mode wrong-key path, and idle animation method defined but never called in the game loop.

</domain>

<decisions>
## Implementation Decisions

### Avatar ID Mismatch (PROF-01)
- **D-01:** Update `DEFAULT_UNLOCKED_AVATARS` in `src/persistence/types.ts` from legacy IDs (`avatar-red`, `avatar-blue`, `avatar-green`) to current IDs (`avatar-kid-01`, `avatar-kid-02`, `avatar-alien-01`).
- **D-02:** No schema migration needed. Existing LocalStorage profiles with legacy IDs are not migrated — acceptable since no real users exist yet. The `LEGACY_ID_MAP` in `definitions.ts` already handles runtime lookups if needed.

### Word Mode Difficulty Signal (DIFF-04)
- **D-03:** Add `this.difficulty.recordResult(false)` in `_processWordInput()` wrong-key branch (states.ts, else block around line 843-846). Mirror the letter mode pattern where every wrong key counts as a miss for difficulty adjustment.
- **D-04:** No change to per-character hit tracking. Only full word completion calls `recordResult(true)` — this remains unchanged. The fix is strictly about the missing miss signal.

### Alien Idle Animation (AV-01)
- **D-05:** Call `entity.container.updateIdle(dt)` in the `PlayingState.update()` loop for ALL active entities — both `activeEntities` (letters) and `activeWordEntities` (words).
- **D-06:** Idle animation runs on all aliens uniformly, including those currently running tweens (hit/miss/escape). The tween system and idle system affect different properties so there is no conflict.

### Claude's Discretion
- Exact placement of `updateIdle()` call within the update loop (before or after tween updates)
- Whether to update existing unit tests or add new ones for the bug fixes
- Any minor code cleanup directly related to the three fixes

</decisions>
