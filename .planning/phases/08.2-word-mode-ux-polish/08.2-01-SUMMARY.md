SplitBitmapText per-character gold tinting on typed characters in word mode, plus pulsing glow highlight on preferred game mode button in MenuState.

## Implementation Details

### Task 1: Per-Character Word Feedback (TDD)

**Type change pipeline:** `WordEntity.wordText` changed from `BitmapText` to `SplitBitmapText` across words.ts, alien-container.ts, and states.ts. SplitBitmapText provides `.chars[]` array for individual character tint control.

**Spawn changes:** `_spawnWord()` creates `SplitBitmapText` instead of `BitmapText`. Uses `pivot.set()` for centering (SplitBitmapText extends Container, has no anchor). Text setter auto-resets chars to white (0xffffff).

**Input feedback:** `_processWordInput()` applies gold tint (0xffd700) to chars[0..cursorIndex-1] on correct keypress. All chars gold on completion. No tint change on wrong key (dodge tween only).

**Pool reuse safety:** `AlienContainer.reset()` iterates `wordLabel.chars` and resets all tints to 0xffffff, preventing gold bleed between reused word entities.

### Task 2: Preferred Mode Glow

**Glow creation:** In `MenuState.enter()`, reads `ctx.getActiveProfile()?.preferredGameMode`. When non-null, creates a `Graphics` rounded-rect stroke (SPACE_PALETTE.glow / 0x6b8bf5, 3px width, 8px padding) positioned over the target panel. Inserted at z-index 0 via `addChildAt()`.

**Animation:** `MenuState.update()` oscillates `glowGraphic.alpha` via `0.4 + 0.4 * Math.sin(glowTime * 2.5)`, producing a gentle 0.0-0.8 alpha pulse at ~0.4 Hz.

**Null handling:** No glow on first launch (preferredGameMode is null) or when no profile exists. Non-preferred button stays unchanged.

**Cleanup:** `MenuState.exit()` nulls glow references; the Graphics object is destroyed as a child of menuContainer.

## Decisions Made

1. **SplitBitmapText over BitmapText for word labels** - Enables per-character tinting via .chars[i].tint without manual character splitting
2. **pivot.set() for centering** - SplitBitmapText extends Container (no anchor property), pivot is the correct centering mechanism
3. **Sinusoidal alpha pulse at 2.5 rad/s** - Produces ~0.4 Hz oscillation, gentle enough for children, non-distracting
4. **addChildAt(glow, 0) for z-ordering** - Places glow behind both panels for clean visual layering
