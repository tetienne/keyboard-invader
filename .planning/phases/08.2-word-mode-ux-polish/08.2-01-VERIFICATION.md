### Human Verification Required

#### 1. Per-character gold tint visible in browser during word mode

**Test:** Start the game, select a profile, choose Words mode, begin a session. Type the first character of a falling word correctly.
**Expected:** The first character of the word turns gold (#FFD700) immediately; the remaining characters stay white. Type the second character — it also turns gold; first and second are now gold, rest white.
**Why human:** PixiJS canvas rendering — `SplitBitmapText.chars[i].tint` mutations are applied to the WebGL/Canvas render pipeline. Unit tests mock the chars array and confirm the tint values are set correctly, but actual visual output requires in-browser inspection.

#### 2. Menu preferred mode glow visible after playing a session

**Test:** Play a complete session in Letters mode (or Words mode). Return to the menu.
**Expected:** The Letters (or Words) button has a visible pulsing glow outline — a rounded-rect stroke in the theme blue-purple color (#6B8BF5) that slowly oscillates in opacity.
**Why human:** GSAP/PixiJS alpha animation — the `Math.sin(glowTime * 2.5)` oscillation produces a visual animation. Unit tests confirm `glowGraphic.alpha` is updated on each update tick, but the rendered visual effect requires browser inspection.

#### 3. No glow on first launch (null preferredGameMode)

**Test:** Clear all profile data (localStorage), reload the app, create a new profile, navigate to the menu without playing any session.
**Expected:** Neither the Letters nor the Words button has any glow or highlight. Both buttons appear in their normal state.
**Why human:** Visual canvas state — unit test asserts `addChildAt` is not called when `preferredGameMode` is null, but the actual absence of any glow on the canvas requires visual inspection.

### Gaps Summary

No gaps found. All 7 must-have truths are verified in code. The 3 human verification items listed above are visual behaviors on the PixiJS canvas that cannot be unit-tested; they require in-browser inspection to confirm the rendering pipeline delivers the intended UX.
