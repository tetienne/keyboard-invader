## Gaps

### Gap 1: SplitBitmapText crash on empty text
status: resolved
found: 2026-04-19 during browser UAT
symptom: Word mode spawned no falling aliens. Console showed `TypeError: Cannot read properties of undefined (reading 'parent')` at `_SplitBitmapText.addChild` → `split` → `onTextUpdate` → `styleChanged` → constructor.
root_cause: `new SplitBitmapText({ text: '', ... })` triggers `styleChanged` → `split()` which calls `this.addChild(...this.lines)`. For empty text, `lines` is empty, so `addChild()` is called with zero args and accesses `undefined.parent`.
fix: Construct `SplitBitmapText` with the actual word instead of empty string. Subsequent pool reuses call `.text = newWord` which handles the intermediate empty state correctly.
