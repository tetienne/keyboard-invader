| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a3e` / `0x1a1a3e` | Deep space background (`SPACE_PALETTE.background`) |

| Glow | `#6b8bf5` / `0x6b8bf5` | Panel borders, **preferred mode glow aura** (`SPACE_PALETTE.glow`) |
| Gold feedback | `#FFD700` / `0xffd700` | **Per-character typed feedback** -- correctly typed characters in word mode |
| White (text default) | `#ffffff` / `0xffffff` | Untyped characters in word mode, default BitmapText color |

### Phase-Specific Color Contracts

**Per-character word feedback (D-01):**
- Typed characters: gold `0xffd700` tint applied to individual `SplitBitmapText.chars[i]`
- Untyped characters: white `0xffffff` tint (default)
- The boundary between gold and white IS the cursor -- no additional cursor indicator
- On word reset/pool reuse: all chars reset to `0xffffff`

**Preferred mode glow (D-05):**
- Glow color: `SPACE_PALETTE.glow` (`0x6b8bf5`)
- Glow rendered as `Graphics.roundRect` stroke, 3px width
- Glow padding: 8px outside the space panel border
- Glow corner radius: `UI_CONSTANTS.panelCornerRadius + 4` (16px)

Accent reserved for: score highlight, destruction particle bursts, critical-action buttons. Never for per-character typing feedback (gold is used instead) or mode selection glow (blue glow is used instead).

Source: CONTEXT.md D-01, D-05; `src/game/theme.ts` SPACE_PALETTE

## Interaction Contracts

### 1. Per-Character Typing Feedback (Word Mode)

**Trigger:** Each correct keypress in word mode (`_processWordInput` result === 'correct' or 'complete').

**Visual response:**
- The character at `cursorIndex` position changes from white (`0xffffff`) to gold (`0xffd700`) tint
- No animation on the character -- instant color change on keypress (D-02)
- All characters from index 0 to `cursorIndex - 1` are gold; `cursorIndex` onwards remain white
- On word completion (`result === 'complete'`), all characters are gold before the hit tween plays

**Implementation surface:**
- `SplitBitmapText` replaces `BitmapText` for word labels
- `SplitBitmapText.chars[i].tint` provides per-character tinting
- Centering: `pivot.set(width / 2, height / 2)` instead of `anchor.set(0.5)` (SplitBitmapText extends Container, not BitmapText)

**States:**
| State | Visual |
|-------|--------|
| Word spawned | All characters white `0xffffff` |
| Partial progress | chars[0..cursorIndex-1] gold `0xffd700`, rest white |
| Word complete | All characters gold, then hit tween + destruction |
| Wrong keypress | No color change on characters; dodge tween on alien container |
| Pool reuse | All characters reset to white on new word assignment |

Source: CONTEXT.md D-01 through D-04; RESEARCH.md SplitBitmapText architecture

### 2. Preferred Mode Glow (Menu)

**Trigger:** `MenuState.enter()` reads `ctx.getActiveProfile()?.preferredGameMode`.

**Visual response:**
- If `preferredGameMode === 'letters'`: glow aura on letter mode panel
- If `preferredGameMode === 'words'`: glow aura on word mode panel
- If `preferredGameMode === null` (first launch): no glow on either panel (D-07)
- Non-preferred panel: normal state, no dimming (D-06)

**Glow animation:**
- Alpha oscillation: `0.4 + 0.4 * Math.sin(time * 2.5)`
- Range: 0.0 to 0.8 alpha (smooth sinusoidal pulse)
- Frequency: approximately 0.4 Hz (one full cycle every 2.5 seconds)
- Driven by `MenuState.update(dt)` via PixiJS ticker (already available)
- Non-distracting for children -- gentle pulse, not a strobe

**Implementation surface:**
- A `Graphics` object drawn as a rounded rectangle stroke behind the preferred mode's space panel
- Inserted at z-index 0 in `menuContainer` (behind panels)
- Destroyed in `MenuState.exit()` with the rest of the menu container

**States:**
| State | Visual |
|-------|--------|
| First launch (null preference) | Both panels equal, no glow |
| Preferred = letters | Glow pulses around letter panel |
| Preferred = words | Glow pulses around word panel |
| Hover on any panel | Existing scale 1.1 feedback unchanged |

Source: CONTEXT.md D-05 through D-08; RESEARCH.md Glow Animation Pattern
