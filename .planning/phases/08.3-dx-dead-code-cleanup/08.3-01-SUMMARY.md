## Accomplishments

- Removed PausedState entirely: class definition (40 lines), `paused: new PausedState()` registration in `Game` constructor, import from `states.js`, re-export from `index.ts` barrel, and the `'paused'` literal from both the `StateName` union and the `TRANSITIONS` map. The `Record<StateName, GameState>` exhaustiveness check stayed valid because both the type and the registration shrank together.
- Live pause flow preserved: `_pause()`, `_resume()`, `_unpause()`, `_showPauseOverlay()`, `_hidePauseOverlay()`, `_pauseOverlay`, `_isPaused` (19 grep matches in `game.ts`) all untouched per D-03.
- Removed dead tween factories `createMissTween` and `createBottomTween` from `src/game/tween.ts` and from the barrel re-export. `LetterTween.type` union literals `'miss'` and `'bottom'` and the corresponding `updateTween` switch branches were kept per D-15 (no scope creep).
- Removed `preallocatePools` method from `Game` and from the `GameContext` interface. Pools allocate lazily on first `acquire()`, so behavior is identical.
- Demoted `resolveLevel` from `export function` to module-private `function`. The internal call from `applyXp` at `progression.ts:70` (`Math.min(resolveLevel(newTotalXp), MAX_LEVEL)`) is preserved.
- Updated all four affected test files to compile cleanly:
  - `tests/game/states.test.ts`: drop `PausedState` import, drop `paused: createMockState()` from `makeStates()`, swap invalid-transition target `'paused'` → `'gameover'` (still throws — `'gameover'` is not in `TRANSITIONS.boot`), delete `describe('PausedState', …)` block, drop `preallocatePools` mock entry from the mock context.
  - `tests/game/tween.test.ts`: drop `createMissTween` + `createBottomTween` imports, delete the two `it(...)` shape tests, delete `describe('miss tween', …)` and `describe('bottom tween', …)` blocks.
  - `tests/game/progression.test.ts`: drop `resolveLevel` import, delete `describe('resolveLevel', …)` block. Transitive coverage by `applyXp` tests preserved.
  - `tests/game/profile-state.test.ts`: drop `preallocatePools` mock entry.
