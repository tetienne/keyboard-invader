## Implementation Decisions

### Pause Architecture (PausedState removal)
- **D-01:** **Delete** `PausedState` entirely. Remove the class definition (`src/game/states.ts:1253`), the import in `src/game/game.ts:20`, the registration in `src/game/game.ts:85` (`paused: new PausedState()`), and the re-export in `src/game/index.ts:7`. Pause continues to use the existing `_pause()` / `_resume()` / `_pauseOverlay` flow in `src/game/game.ts` — that path is **not** dead.
- **D-02:** Do **not** rewire pause to transition through the FSM. GAME-06 verification already passes; refactoring pause architecture is out of scope for a cleanup phase.
- **D-03:** Leave `_pauseOverlay` and its child elements (Graphics bg, text, hint) untouched — they are used by the working pause flow.

### commit-msg Hook Setup (DX-01)
- **D-04:** Update `README.md:16` install command from `prek install` to `prek install --hook-type pre-commit --hook-type commit-msg` so fresh clones install both hooks.
- **D-05:** End-to-end verification: attempt a commit with a non-conventional message (e.g., `git commit -m "bad message"`) and confirm prek/commitlint rejects it via the local `.git/hooks/commit-msg` hook.
- **D-06:** Do **not** add a CI-level commitlint job. The phase SC requires the hook validates "on every commit" — local hook + correct README satisfies that. CI commitlint is a separate concern.

### Dead Exports Removal
- **D-07:** Remove from `src/game/tween.ts`: `createMissTween` (line 25) and `createBottomTween` (line 30). Replaced by escape/dodge tweens earlier; no remaining call sites.
- **D-08:** Remove from `src/game/progression.ts`: the `export` keyword on `resolveLevel` (line 52) — it is only used internally within the module by `addXp`. Keep the function itself, demote to module-private.
- **D-09:** Remove from `src/game/game.ts:221` and `src/game/types.ts:84`: the `preallocatePools()` method on `GameContext` and its interface declaration. Never called anywhere.
- **D-10:** Update `src/game/index.ts` (lines 36-37 and any others) to drop the corresponding re-exports for the removed/demoted symbols.

### Dead i18n Keys (audit-listed only)
- **D-11:** Remove from **both** `src/shared/i18n/en.json` and `src/shared/i18n/fr.json` exactly the 6 keys flagged by the audit: `app.title`, `app.subtitle`, `boot.loading`, `menu.play`, `menu.profiles`, `menu.settings`. Do not sweep beyond this list — false-positive risk for dynamically-accessed keys is not worth it for this surgical phase.
- **D-12:** Add a **vitest unit test** (a new file under `tests/i18n/` or co-located with existing i18n tests) that loads `en.json`, iterates each key, and asserts at least one match for that exact key string under `src/` (excluding the i18n JSON files themselves). This guards against future dead keys regressing in.
- **D-13:** The test reads `en.json` as the source of truth for keys (assume `fr.json` mirrors it — a separate parity check is out of scope).

### Vitest Config
