## Integration Check Results

### Critical Defects

**1. DEFAULT_UNLOCKED_AVATARS Legacy ID Mismatch (PROF-01)**
- **Severity:** Critical (functional regression)
- **Location:** `src/persistence/types.ts` — `DEFAULT_UNLOCKED_AVATARS` contains `['avatar-red', 'avatar-blue', 'avatar-green']`
- **Issue:** `AVATARS` in `definitions.ts` uses new IDs (`avatar-kid-01`, `avatar-kid-02`, `avatar-alien-01`). New profiles created after Phase 8's avatar ID rename will have all avatars appear locked because `unlockedAvatarIds.includes(def.id)` compares mismatched ID schemes.
- **Fix:** Update `DEFAULT_UNLOCKED_AVATARS` to `['avatar-kid-01', 'avatar-kid-02', 'avatar-alien-01']`
- **Affects:** PROF-01

**2. AlienContainer.updateIdle() Never Called (AV-01)**
- **Severity:** Medium (UX quality gap)
- **Location:** `src/game/alien-container.ts:32` — method defined, 0 call sites in codebase
- **Issue:** Aliens fall straight down with no bobbing or blinking animation. The "expressive characters" aspect of AV-01 is partially undermined.
- **Fix:** Add `entity.container.updateIdle(dt)` in PlayingState update loop (~2 lines)
- **Affects:** AV-01

### Integration Findings

**3. Word Mode Difficulty Signal Incomplete (DIFF-04)**
- **Location:** `src/game/states.ts` ~line 845
- **Issue:** Wrong-key presses in word mode increment `this.misses` but do not call `this.difficulty.recordResult(false)`. The DifficultyManager's rolling window never sees wrong-key-presses in word mode — only word escapes and completions register. Adaptive difficulty in word mode responds only to word-level events.
- **Affects:** DIFF-04 (word mode only)

**4. PausedState Permanently Unreachable (GAME-06)**
- **Issue:** `PausedState` registered in state machine but no code calls `transitionTo('paused')`. Pause works via direct ticker stop in `Game._pause()`. Dead code.
- **Affects:** GAME-06 (feature works, implementation is non-idiomatic)

**5. Per-Character Typing Feedback Absent in Word Mode**
- **Issue:** `cursorIndex` is tracked internally but no visible per-character color change during typing. Children cannot see which characters they've already typed correctly.
- **Affects:** GAME-03, GAME-04 (UX quality for word mode)
